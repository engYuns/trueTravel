## Implementation Guidelines

### DO ‚úì
- Use SVG icons with `fill="currentColor"`
- Keep icon size consistent at `w-5 h-5` (20x20px)
- Use Material Design icon paths
- Apply color through parent element's text color
- Use semantic icon choices (flight for aviation, calendar for reservations, etc.)

- Don't use emoji icons (‚ùå ‚úàÔ∏è üè® üìÖ)
- Don't use inconsistent icon sizes

## Adding New Icons

When adding new icons to the platform:

1. **Find Material Design Icon**: Use [Material Design Icons](https://fonts.google.com/icons) or similar
2. **Extract SVG Path**: Copy the `<path d="..."/>` data
3. **Apply Standard Structure**:
```tsx
<svg className="w-5 h-5" fill="currentColor" viewBox="0 0 24 24">
  <path d="[PASTE PATH DATA HERE]"/>
</svg>
```
4. **Test Color Inheritance**: Ensure `fill="currentColor"` works with different text colors
5. **Document**: Add the new icon to this guide

## Color Usage

Icons automatically inherit color from parent element:
- **Active state**: `text-orange-500` (orange)
- **Inactive state**: `text-white` or `text-gray-600`
- **Hover state**: `hover:text-orange-500`

Example:
```tsx
<a className="text-white hover:text-orange-500">
  <svg className="w-5 h-5" fill="currentColor" viewBox="0 0 24 24">
    <path d="..."/>
  </svg>
</a>
```
